Cybersecurity Internship Report Summary

Learn about the defensive algorithms, concepts, and interview questions related to this project

Shannon Entropy

Measures the uncertainty of a password. Defined mathematically as \(E = L \times \log_2(R)\), where \(L\) is the password length and \(R\) is the character pool size. Incorporating digits, casing, and special symbols increases \(R\), raising brute-force cracking difficulty exponentially.

Dictionary Attacks

Attackers use wordlists of precomputed weak passwords. Our application uses a separate blacklist database loaded dynamically to check and immediately block these vulnerability entries.

OSINT & Social Engineering

Users commonly structure passwords using public details like email usernames. Rejection of email-derived strings (e.g. splitting `abc22@gmail` and blocking `abc`, `22`, `abc22`) counters profiling vectors.

Repeating Pattern Checks

Flags identical consecutive character lines (like `aaaaaa`) and sliding pattern loops (like `12121212` or `abcabcabc`) to protect against custom pattern-guessing tools.

Sequential Scans

Checks character windows of length 5 against alphabetical lists, digits, and keyboard layouts (`qwertyuiop`, `asdfghjkl`, `zxcvbnm` and reverses) to reject easy keyboard walks.

Modular Design

The code segregates user interfaces from the checking logic. This architecture allows developers to swap, scale, or integrate checks easily (e.g. calling HaveIBeenPwned API).

Q1: Why reject a password immediately instead of just assigning a low score?

A: Severe structural flaws (like length < 8, blacklist matches, or keyboard walks) are vulnerabilities so extreme that character variety cannot compensate. Blocking them immediately prevents a false sense of security.

Q2: How does a special character or number increase resistance mathematically?

A: By expanding the character pool (\(R\)) from 26 (lowercase letters only) to 94 (alphanumeric and symbols), the total pool combinations for an 8-char password increases from \(26^8 \approx 2 \times 10^{11}\) to \(94^8 \approx 6 \times 10^{15}\), making brute-force cracking exponentially slower.

Q3: How do attackers exploit email-derived credentials?

A: In credential stuffing attacks, bots combine usernames found in leaked databases with standard modifiers (like appending years or special characters) to guess passwords. Email-derived filtering neutralizes this.
